Transformations

    Transforms

  • The transform property allows you to translate, rotate, scale and skew any element on the page. This is a static, one time change.
  • Transformations are nice because they only move the element, they don't cause the rest of the document to change.
  • Only declare on transform per element. Additional ones will overwrite the previous entries.
  • Translation allows you to move elements left, right, up and down.
  • Translations do not work on "inline" elements. Change inline elements to "inline-block" for them to function.
  • Scaling will grow or shrink an element along each axis.
  • Rotation will rotate the by degrees, grads, radians or "turns" on the z-axis.
  • Skewing will slant the element along each axis.
  • Elements will grow/shrink and rotate from its center point by default.
  • You can change the transformation origin with the transform-origin property. This allows you to rotate an object around the a different point on the element like the top, so it looks like the element rotates around the top of the element.
  • The order of transforms is important! Translating then rotating will have a very different affect than rotating then translating

    • Transitions

  • Transitions are a way to animate visual changes to an element, include transformations.
  • They allow the value of CSS properties to change over time.
  • Basic steps to creating a transition.
    1. Declare the original state of the element in the default style declaration.
    2. Declare the final state of your transitioned element; for example, a :hover state.
    3. Include the transition functions in your default style declaration using the transition properties, including: transition-property , transition-duration , transition-timing-function , and transition-delay . Weâ€™ll look at each of these and how they work shortly.
  • The important point to note is that the transition is declared in the default or originating state.
  • The transition-property property declares all of the CSS properties of the element that should be transitioned. The all keyword for all properties is the default.
  • The transition-duration property takes the value for how long an animation should take in either seconds (s) or milliseconds(ms)
  • Most animations should take about 200ms. Going slower can make the site feel sluggish, going faster can go unnoticed.
  • The transition-timing-function property allows you to vary the pace of the animation from start to end. For example, you can make the animation start slow and speed up towards the end.
  • The transition-delay property can cause the animation to delay play for a set amount of time.
  • All of the transition properties can be combined on a single line.
  • For transitions with multiple phases, comma separate pairs of properties

Animations

  • Animations allow more control than transitions via Keyframes
  • Keyframes are multiple steps through an animations.
  • The Keyframe keyword "from" and "to" represent the start and end of the animation or 0% and 100% of the animation time
  • You can define as many keyframes as you need.
  • The animation-name property is use to attach an animation to an element.
  • The animation-duration property defines how long the animation takes to complete.
  • The animation-timing-function property allows you to vary the pace of the animation from start to end. For example, you can make the animation start slow and speed up towards the end
  • The animation-iteration-count property dictates how many times to loop through the animation. The value of "infinite" is a continuous loop.
  • The animation-direction property say if the animation runs forward or in reverse. The value of alternate make the animation run forwards then in reverse.
  • The animation-delay property will make the animate wait after triggered for a given time frame before playing.
  • The animation property is the shorthand version that will let you set all of the properties on a single line.

Canvas

  • Canvas is a way to draw 2d vector graphic on a web page.
  • The Canvas element is the only element needed to work with canvas. The remainder of the work is done in JavaScript.
  • Set the height and width attributes in the canvas element to set the actual size of the canvas object. The CSS properties only change the size of the containing box.
  • The "context" is where the drawing is rendered.
  • Set both the strokeStyle and fillStyle properties on the context before drawing anything.
  • Canvas's basic coordinate system starts with a point on the top left and ends with a point on bottom right.
  • The fillStyle can be solid colors, gradients and images.
  • When using images, apply the image to the canvas using the images "onload" function to make sure the image is downloaded before attempting to use it.
  • Linear Gradients are define with the top/left and bottom right coordinates and two or more "ColorStops" to help define the color migration
  • Shapes that aren't rectangles are created using "paths"
  • A circle can be created using an "Arc" which takes X and Y coordinates, radius, start and end angle, and direction of rotation (i.e. clockwise).
  • Note: 1 radian is 180 degrees or half a circle.
  • Arc must also have a stroke or fill style to be seen.
  • Canvas drawing can be save as images, in png format.
  • Images drawn with canvas can be manipulated, such as the color of each pixel can be changed. For example you can turn a color image into a black and white one.
  • Video can all be drawn and manipulated in canvas. For example you can overlay text on the video.
  • Canvas has no accessability features and will be invisible to screen readers.

SVG

  • SVG stands for Scalable Vector Graphics, which are recorded in an XML format, and are another way to draw on a web page.
  • SVG can be seen by screen readers.
  • SVG has more basic shapes available via dedicated functions, like circles.
  • It is usually better to make SVG's in a 3rd party editor like Illustrator, than to code them manually.
  • You can modify the properties of SVG via CSS and Javascript.

Drag and Drop

  • This feature lets you move elements on the web page with your mouse or finger.
  • It can be used in combination with the File API to drag file from a folder onto the web page for usage or upload.
  • There are three steps to enabling drag and drop.
    • Set the "draggable" attribute on the HTML elements to be mobile to "true"
    • Add the event handlers for "dragstart" to all elements you want to drag.
    • Add the event handlers for "dragover" and "drop" events on any elements that will take the dropped items.
  • The DataTransfer object can be used to hold and retrieve information about the dragged elements. For example, the Id of the entity being moved, not the HTML element.
