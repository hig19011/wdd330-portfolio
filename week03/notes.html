this

  • The "this" keyword is needed to access members of the parent object from inner scopes because of inherent deficiencies in the language.
  • "this" attempts to make an Javascript objects act more like classes in well structured languages.
  • Be careful when using "this", its behavior is inconsistent across the language.
  • If a function isn't part of an object, using the "this" property will still be consider valid even though it will fail at runtime.
  • Learn to use "this", it isn't going changing or leaving the language

Objects

  • Object are basically untyped dictionaries that can hold primitives, other objects and even functions
  • Create a new object with the following syntax "const person = {};"
  • Access properties with either of the following syntax person.name; person["name"];
  • Check to see if a property exists on an object like this person.name !== undefined
  • New properties can be added at any time by assigning a value to a new property
  • A property can be removed by using the delete keyword delete person.address;
  • Object can be nested, meaning objects can have properties that are also objects
  • Objects are copied by reference
  • Namespacing is simulated by putting similar objects in another object

      • Built in Objects or libraries

    • JSON - for generating and parsing JSON
    • Math - general math functions
    • Date - generic date handling object - can't deal with timezones
    • RegEx - pattern matching library using regular expressions

DOM - Document Object Model

  • The DOM is the html for the page.
  • The DOM is structured as a tree where parent nodes have zero to many children.
  • The DOM is accessed via the document object.

      • Shortcuts

    • document.body returns the body element of the page
    • document.images returns a list of images on the page
    • document.links returns a list of and elements with an href
    • document.anchors returns a list of elements with a name
    • document.forms returns a list of all forms on the page

      Getting elements

    • document.getElementById('title'); returns the single element with that id
    • document.getElementsByTagName('li'); returns all elements with that tag name
    • document.getElementsByClassName('hero'); returns all elements with that include that class
    • document.querySelector('#bats'); returns all elements that would be affected by that CSS selector

  • myDiv.children returns all direct children of the element in myDiv
  • myDiv.firstChild returns the first child of the element in myDiv
  • myDiv.lastChild returns the last child of the element in myDiv
  • myDiv.nextSibling returns the last child of the element in myDiv
  • use setAttributes and getAttributes to change attributes on an element
  • use myDiv.classList.add/remove to add or remove a class from an element
  • use myDiv.toggle("bold") to add a class to an element if it is missing or remove it if it has the class
  • document.createElement('li'); will create a new element, a listitem in this case
  • myDiv.appendChild(myLi) will add the listitem "myLi" to "myDiv"s children list
  • myDiv.removeChild(myLi) will remove the listitem "myLi" from "myDiv"s children list
  • myDiv.innerHTML will return all the elements children as a string of HTML
  • a - separated name like background-color can be access via dot notation when you remove the - and use camel casing myDiv.backgroundColor

Events

  • Events notify you when something occurs instead of you having to constantly poll to see if something happened.
  • Event listeners know when the event happened and will let you know if you register with them.
  • Event registration example document.body.addEventListener("click", doSomething);
  • Inline event handler at an attribute on an element onclick="console.log('You Clicked Me!')"
  • The event object is the parameter that is passed into to the even handler function doSomething(event) { console.log(event.type); }
  • The event object tell you information about the event like which element was clicked, what the mouse coordinates were, what key was pressed
  • Remove an event handler by doing this myDiv.removeEventListener('click',remove);
  • You can prevent the default behavior by doing this event.preventDefault(); in the event handler
  • Event bubbling is the default way events are handled a clicked event bubbles up from the child to the parent up through the DOM tree
  • Event Capturing makes the even fire on the top parent first, then propagating down to the children
  • Event delegation is when you attach a event handler to a parent element to capture event triggered by its children.
